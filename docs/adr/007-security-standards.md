# ADR-007: 서비스 보안 설계의 상위 참조 기준 채택

## Status

Accepted (2026-05-26)

## 결정 요약

본 문서는 SOC 2, ISMS-P, 개인정보 보호법령 및 안전성 확보조치 기준을
서비스 보안 설계의 상위 참조 기준으로 사용한다.

다만 본 문서는 조직 운영, 인증 심사, 내부 감사, 상시 보안 운영 체계 전체를
다루지 않는다. 대신 각 기준에서 요구하는 통제 목표를 서비스 기능, 데이터 흐름,
접근통제, 감사로그, 설정 변경, 개인정보 처리 방식에 어떻게 반영할지 정의한다.

## Context

`llmgate`는 OpenAI wire-compatible LLM router이자 호출자 인증 경계다. 서비스는
consumer bearer key, provider API key, prompt/message/tool payload, LLM 응답,
IP/User-Agent가 포함될 수 있는 감사 로그, 선택적 `llm.result.finalized` 이벤트를
처리한다. 따라서 보안 기준은 단순한 운영 체크리스트가 아니라 서비스 설계의 일부여야 한다.

동시에 SOC 2와 ISMS-P는 조직 운영, 인사/물리보안, 감사기간 증거, 인증 심사
절차까지 포함하는 넓은 프레임이다. 이 범위를 그대로 `llmgate` 리포지터리에
가져오면 서비스 코드가 책임질 수 없는 항목까지 필수 요구사항으로 섞인다.

## Decision

`llmgate`의 보안 설계는 다음 기준을 상위 참조로 사용한다.

| 기준 | 서비스 설계에서 사용하는 방식 |
|------|-------------------------------|
| 개인정보 보호법령 및 개인정보의 안전성 확보조치 기준 | LLM 요청/응답에 개인정보가 포함될 수 있음을 전제로, 서비스가 직접 제공하거나 보조 증적으로 생성할 수 있는 접근통제, 암호화 전제, 접속기록 보조 필드, 보존/파기 경계, 침해 조사 증거 기준으로 사용한다. |
| ISMS-P | 전체 인증 범위가 아니라 서비스 기능에 적용 가능한 보호대책 및 개인정보 처리단계 요구사항만 선별해 사용한다. |
| SOC 2 | 감사 보고서 준비가 아니라 Security와 Confidentiality 중심의 서비스 통제 설계와 증거 생성 기준으로 사용한다. |

서비스 기준에 포함하는 항목은 다음으로 제한한다.

- 코드로 강제 가능한 보안 동작
- 환경변수/설정으로 fail-fast 또는 보안 기본값을 제공할 수 있는 항목
- `catalog/`, `consumers/`, audit log, metrics, result event처럼 서비스가 증거를 생성하는 항목
- 테스트로 회귀를 막을 수 있는 보안 계약

서비스 기준에서 제외하는 항목은 다음과 같다.

- 조직 운영, 내부 감사, 인증 심사 대응
- 인사보안, 물리보안, 외부자 계약, 공급망 심사
- 운영자 접근권한 정기 리뷰, 백업 복구 훈련, 상시 관제 체계
- 개인정보 처리방침 게시, 약관/동의 UX, 정보주체 민원 처리
- SOC 2 보고서와 ISMS-P 인증서 자체

구체적인 적용 기준은 [docs/security/00-standards.md](../security/00-standards.md)가
맡고, 현재 구현 상태와 gap은 [docs/security/01-current-state.md](../security/01-current-state.md)가
맡는다.

## Consequences

### Positive

- 보안 기준이 서비스 코드와 문서에서 실제로 적용 가능한 요구사항으로 좁혀진다.
- SOC 2, ISMS-P, 개인정보 보호법령의 통제 목표를 서비스 설계 언어로 재사용할 수 있다.
- 운영 조직이 해야 할 항목과 `llmgate` 리포가 해야 할 항목이 분리된다.

### Negative

- 이 ADR만으로 SOC 2 감사나 ISMS-P 인증을 준비할 수는 없다.
- 운영 증거가 필요한 고객/감사 대응은 별도 문서와 프로세스가 필요하다.
- 서비스 밖 책임을 호출 앱과 운영 조직이 명확히 받아야 한다.
