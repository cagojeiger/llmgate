# ADR 005: 타임아웃 권위자 — 단계당 한 컴포넌트

- Status: Accepted
- Date: 2026-05-03
- 관련: 001, 004

## 문제

한 호출에 timer 가 겹치면 *어느 한도가 fail 을 만들었는가* 가 흐려지고, 동시 살아있는 timer 가 많을수록 cancel race 표면이 곱연산으로 늘어난다.

## 결정

요청 생애주기를 시간축으로 자르고 각 구간의 권위자를 한 컴포넌트로:

- **요청 총 wall-clock** → Handler. 호출 진입에서 1 회. 모든 단계가 이 안에 산다. 호출자에 노출되는 상한.
- **non-stream 시도당** → llmrouter.Service. 시도가 끝나면 즉시 해제. 폴백 트리거의 자연 반차원이라 ADR 004 와 같은 컴포넌트.
- **스트림 idle (이벤트 사이)** → streamRelay. 누적 한도 아님 — 긴 응답 자연 통과.
- **스트림 첫 이벤트 별도 한도 *없음*** — 어댑터가 호출 ctx 만 의지. 별도 timer 는 race 곡예 비용.
- **회로 차단 시간 / 백오프** — 시간 *간격* 이지 시간 *한도* 가 아님. ADR 004 영역.

## 이유

- 동시 살아있는 timer ↓ → race 표면 ↓.
- `record.error_kind` 가 컴포넌트 책임으로 즉시 매핑 (Handler / Service / streamRelay).
- 한도 추가 요청에 같은 잣대 — *어느 단계 권위자에 둘 것인가* 부터.

## 대가

- 운영자가 3 한도 (총 / 시도 / idle) 를 머리에 둠 — 권위자가 컴포넌트로 묶여 추적 짧음.
- 첫 이벤트 도착 한도 별도 없음 → 총 한도 길게 잡으면 첫 이벤트 전에도 그 한도까지 기다림. 운영자는 총 한도를 *첫 이벤트 commit* 으로 봄.
