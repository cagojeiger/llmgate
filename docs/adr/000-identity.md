# ADR 000: 호출자는 모델 호출 인프라를 관리하지 않는다

- Status: Accepted
- Date: 2026-05-08
- 관련: 001 ~ 006 (이 원칙에서 파생)

## 문제

호출자 코드가 LLM을 쓰면 task와 무관한 인프라가 따라 들어온다 — 벤더 SDK 선택, 키 관리, 폴백, 회로 차단, 와이어 변환. 호출자마다 같은 인프라를 다시 짜고 또 짠다.

또 *어떤 모델을 쓸지*가 호출자 코드에 박힌다. task 의도("복잡한 추론", "짧은 분류", "코드 작성")를 표현하려고 호출자가 구체 모델 ID(`deepseek-v4-pro`, `qwen3.6-plus` 같은 것)를 들고 있어야 하고, 모델 정책을 바꾸면 호출자 N개를 동시에 고쳐야 한다.

게다가 내부 운영용 게이트웨이라 multi-tenant SaaS 기능은 비용일 뿐 가치가 아니다.

## 결정

**호출자는 OpenAI SDK를 그대로 쓰고, 구체 모델 ID 대신 `smart`, `worker`, `cheap` 같은 alias만 보낸다.** 호출자는 "이 task에 어떤 성격의 모델이 필요하다"만 표현하고, 실제 어떤 벤더의 어떤 모델을 쓸지는 게이트웨이가 카탈로그를 보고 정한다.

**운영자는 alias와 ordered chain을 YAML로 관리한다.** `smart`가 어떤 chain인지, `cheap`이 어떤 우선순위를 갖는지는 운영자 정책이다. 게이트웨이는 그 의미를 해석하지 않고, alias를 chain의 식별자로 보고 등록된 순서대로 모델 호출을 시도한다.

**게이트웨이는 작은 실행기로 남는다.** 다음만 흡수한다:

- 벤더별 wire 차이 (벤더 SDK 대신 직접 wire adapter)
- 벤더 키 관리
- 폴백 체인 (ordered chain, weighted / round-robin / latency-aware 라우팅 X)
- 반복 실패한 모델의 임시 차단
- 요청 한 건당 audit record 한 줄을 *밖으로* 발행

**게이트웨이는 SaaS 제어 plane이 되지 않는다.** 다음은 *하지 않는다*:

- 사용자별 과금, rate limit, 사전 quota 차단
- capability discovery (`/v1/models`), 모델 가격·context window 메타 관리
- 모델 적합도 판단, mid-stream 폴백, 멀티키 스마트 분배
- DB·Redis 같은 영속 저장소

이 판단들은 게이트웨이 밖의 운영 정책이나 후처리 시스템이 맡는다.

### 경계선

오해를 피하기 위해 세 가지를 명시한다.

- **OpenAI 와이어 호환의 범위는 모델 호출 요청·응답에 한정한다.** OpenAI API의 전체 표면을 구현한다는 뜻이 아니다.
- **YAML 검증 책임.** 게이트웨이는 YAML의 구조적 유효성을 검증한다 — 존재하지 않는 모델 참조, 빈 chain, 잘못된 provider 설정은 부팅 실패. 다만 alias 이름의 의미나 chain 순서의 타당성은 판단하지 않는다.
- **호출자 책임 파라미터.** 호출자는 task 수행에 필요한 요청 파라미터(토큰 한도, 온도 등)를 본인이 결정한다. 벤더 차이, 폴백 동작, 회로 상태는 모른다.

## 근거

- **호출자의 본분은 task 수행이다.** 벤더 SDK 선택, 폴백, 키 관리, wire 변환은 호출자마다 반복할 일이 아니다.
- **task 의도와 모델 ID가 분리된다.** 구체 모델 ID가 호출자 코드에 박히면 모델 정책이 바뀔 때마다 호출자 N개를 동시에 고쳐야 한다. alias를 쓰면 호출자는 task 의도만 표현하고, 실제 모델 선택은 운영자가 카탈로그에서 바꾼다 — 한쪽이 낡아도 다른 쪽이 같이 낡지 않는다.
- **게이트웨이가 작게 유지된다.** 정책을 판단하는 시스템이 아니라 운영자가 정의한 chain을 실행하는 얇은 실행기로 남는다. 한 사람 머리에 들어오는 운영 면적.

## 결과

- 새 모델·벤더 추가는 카탈로그 YAML 변경으로 끝난다. 호출자 코드 변경 0.
- 호출자 코드는 OpenAI SDK와 alias만 알면 된다.
- 모델 폴백, 회로 차단, 벤더 wire 차이는 게이트웨이 안에 머문다.
- 카탈로그 변경은 hot-reload하지 않고 재시작으로 적용한다.
- 비용·quota·capability 판단은 게이트웨이 밖에서 한다.
- 외부 SaaS 시나리오는 디폴트 거절.
